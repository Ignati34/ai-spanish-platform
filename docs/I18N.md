# Internationalization (native languages)

Interface languages: **RU, UK, AR (RTL), FR**, plus ES and EN.

## Frontend
- `apps/web/src/i18n.ts` holds all locale resources and the language switcher wiring.
- Arabic triggers `dir="rtl"` on `<html>` via `applyDir()`; the chosen locale is persisted.
- Add a language: add a block to `resources` and an entry in `LanguageSwitcher`.

## Backend
- `User.native_language` / `interface_language` are captured at registration and Telegram auth.
- Grammar explanations are stored per language in lesson `content_json.explanations_i18n`
  (`ru/uk/ar/fr/es/en`), so the Grammar Agent can answer in the learner's own language.
- `SUPPORTED_LOCALES` / `RTL_LOCALES` in config keep front and back in sync.

## Why per-native-language explanations matter
Different L1 speakers make different Spanish mistakes (ser/estar and articles for RU/UK;
gender and p/b for AR; false friends and r/j for FR). Storing explanations per language lets
the tutor target these instead of giving one generic explanation.
